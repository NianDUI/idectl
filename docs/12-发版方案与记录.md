# 12 · 发版方案与记录

> 把插件打成 zip 并发布到 `NianDUI/idectl` 的 Releases。**本仓库无 CI**，发版是手动的。
> 参考 `/Users/lyd/WorkSpace/MyProjects/7zip` 的 `docs/07-release.md` + `release.sh` 方案：
> 「本地脚本 = CI 步骤的本地等价物」。区别：产物是 **IntelliJ 插件 zip**（非 dmg），无签名/公证问题。

---

## 0. 目标与边界

- **产物**：`build/distributions/idectl-<产品版本>.zip`（`./gradlew buildPlugin` 产出）。
- **触发**：**手动**跑 `./idectl.sh release <tag>`（无 CI，推 tag 不会自动构建）。
- **安装方式**：用户下载 zip → Settings → Plugins → ⚙ → Install Plugin from Disk → 重启 IDEA。
- tag 用 **`vX.Y.Z`**（仓库无上游 tag 冲突，不像 7zip 需要 `mac-v` 前缀）。

## 1. 关键决策（附理由）

| 决策 | 取值 | 理由 |
|---|---|---|
| 发版方式 | **本地脚本**，非 CI | 构建秒级、本地一条命令即可；仓库无 self-hosted runner，搭 workflow 不划算 |
| tag 前缀 | **`vX.Y.Z`** | idectl 仓库无上游版本 tag，裸 `v` 前缀语义清晰无歧义 |
| 签名 | **不适用** | IntelliJ 插件 zip 无 Gatekeeper/公证问题；从磁盘安装即可 |
| 更新说明 | **annotated tag 的 message** | 打 tag 时 `-m` 写「本次更新」，`release.sh` 自动读出拼进 body；贴合「tag = 发版」习惯 |
| 上传 | `gh release create`（Claude 进程可直接跑） | `gh auth status` = 已登录 `NianDUI`（keyring），Bash 进程可直接调用 |

**三条版本线**（别混）：
1. **产品版本** = `gradle.properties` 的 `pluginVersion`（当前 `1.0.0`）→ 注入 zip 文件名 `idectl-<版本>.zip`
2. **发版 tag** = `vX.Y.Z`（GitHub Release 的标识；应与产品版本一致，脚本会软检查）
3. **目标平台下限** = `build.gradle.kts` 的 `sinceBuild`（当前 `261` = IDEA 2026.1）；`release.sh` 自动读出注入 body（`> 目标平台：IntelliJ IDEA 2026.x（sinceBuild XXX…）`），零维护——跟随代码自动跟变。是 7zip「内核版本」的对应物。

## 2. 工具与文件

| 文件 | 作用 |
|---|---|
| `idectl.sh build` | `./gradlew buildPlugin` → `build/distributions/idectl-<版本>.zip`（纯产物） |
| `scripts/release-body.md` | 固定的安装 + 接入说明（跨版本复用） |
| `scripts/release.sh` | **一键发版**：构建 zip → 算 SHA256 → 读产品版本/sinceBuild → 读 annotated tag 的「本次更新」→ 组装 notes → `gh release create` |
| `idectl.sh release <tag>` | 上面脚本的入口封装（保持单一入口，等价于 `bash scripts/release.sh <tag>`） |

## 3. 发版流程

```bash
# 1.（可选）升版本：改 gradle.properties 的 pluginVersion，让它与 tag 一致
# 2. 提交所有改动并推 main
git tag -a v1.0.0 -m "本次更新：修了 XXX、加了 YYY"   # annotated tag，-m 写更新内容（可多个 -m 分段）
git push origin v1.0.0
./idectl.sh release v1.0.0                           # = bash scripts/release.sh v1.0.0
```

`release.sh` 自动完成：`buildPlugin` → 算真实 SHA256 → 读 `gradle.properties` 产品版本 + `build.gradle.kts` `sinceBuild` → 组装 body（`> 目标平台…` + `## 本次更新` + 固定安装/接入说明 + `## 校验`）→ `gh release create v1.0.0 <zip> --title "idectl v1.0.0" --latest`。

- 忘了 `-m`（或打 lightweight tag）也能发，只是 body 没有「本次更新」段，不会误取 commit message。
- 升版本先改 `gradle.properties` 的 `pluginVersion`，`release.sh` 会软检查 zip 版本与 tag 版本是否一致。

## 4. 约束

- 仓库**无 CI**（无 `.github`），推 tag 不触发任何 workflow。
- 发布的 zip 仅在 **IntelliJ IDEA 2026.x** 加载（`sinceBuild = 261`，无向后兼容）。
- 发版前确保 `gradle.properties` `pluginVersion`、tag、（如需）plugin 变更日志三者一致。
