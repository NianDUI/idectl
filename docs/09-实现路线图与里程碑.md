# 09 · 实现路线图与里程碑

> **排期原则（2026-07 定稿）**：用户核心需求——①重新运行正在运行的程序（run/debug 可切换）②检索正在运行程序的控制台日志——必须在 **M1** 完整交付；构建、调试器、Maven、完整治理按需后置。每个里程碑有可验收的 exit criteria，验收不过不进下一阶段。

## 0. 总览

```
M0 骨架(1.5周) ──► M1 核心:会话+控制台+重启(3~4周) ──► M2 构建(3周) ──► M3 多用户治理(2~3周)
                                                              │
              M6 加固与发布(2~3周) ◄── M5 Maven(2~3周) ◄── M4 调试器(3~4周)
```

依赖关系：M1 依赖 M0 的服务器与注册表骨架；M2 的 OperationRegistry 被 M5 复用；M3 治理覆盖此前全部工具；M4/M5 顺序可对调（按届时需要）。总计约 15~18 周到 GA；**M1 结束即可日常自用**。

## M1 · 核心：执行会话 + 控制台检索 + 重新运行（3~4 周）★用户核心需求

| 项 | 内容 |
| --- | --- |
| 范围 | ExecutionRegistry（projectListeners 声明式订阅 EXECUTION_TOPIC；processStarting 挂 ProcessListener 零丢失；**用户手动启动的会话全跟踪**；attachedLate 兜底；owner 归属）；ConsoleRingBuffer（ANSI 剥离、10k 行/8MB、offset/gap、全局 LRU）；工具：`list_run_configurations`、`run_configuration`（wait=none\|exit\|ready + ready_pattern + tail）、`list_sessions`、`stop_session`（soft→force）、**`restart_session`（mode=same\|run\|debug）**、**`console_read`（含 wait_for_pattern/exit 长轮询）**、**`console_search`（ERE+上下文+deadline 熔断）**；单实例冲突预检；单 token（admin 语义）先行 |
| 验收 | ①用户手点绿色按钮启动 Spring Boot → Agent `list_sessions` 看到（startedBy=user）且**首行日志可读**（零丢失时序测试）；②正在 run 的 Application → `restart_session(mode=debug, wait=ready, ready_pattern="Started")` 一次调用完成"优雅停→debug 重启→返回就绪 tail"；③`console_search("ERROR\|Exception", after=3)` 在 10k 行缓冲 P95<100ms，恶意正则 500ms 熔断；④offset/gap 语义在 20 万行压测下正确；⑤已终止会话 30min 内日志可读、exitCode 正确（含 137/SIGKILL 场景）；⑥Gradle 委托运行的会话实测跟踪有效 |

## M2 · 构建域（3 周）

| 项 | 内容 |
| --- | --- |
| 范围 | `build`（四语义合一）、双通道诊断（BuildViewManager+CompilerTopics，1 基统一）、每项目构建队列/互斥（before-launch 并锁）、OperationRegistry + `get_operation`/`cancel_operation`、`get_build_status`（含手动构建感知）、构建输出入缓冲（复用 console_read/search）、`get_test_results`（SMTRunnerEventsListener）、**`reload_classes`（热加载 ★核心③：重编译修改类 → HotSwapUI 热替换进正在 debug 的 JVM，免重启）** |
| 验收 | JPS 与 Gradle 委托两种设置下 build 均返回一致结构化 file:line(1基) 错误且与 IDE Problems 一致；并发 build 第二个观测到 queued+position 并经 get_operation 等到结果；EDT 零阻塞（线程断言测试）；改错→build→按 errors 修复→build 通过 的闭环脚本全自动跑通；JUnit 失败用例经 get_test_results 拿到失败堆栈；debug 运行中改方法体 → `reload_classes` 单次调用热替换生效（**进程 pid 不变**、新逻辑立即可见），改签名/增删方法或类 → 结构化 failed + 建议 `restart_session(mode=debug)`，非 debug 会话 → NOT_DEBUG_SESSION |
