# 04 · 多用户多角色与安全设计

> "多用户" = 多个 Agent 客户端（不同 Claude Code 会话 / Codex / 自动化系统）并发连接同一 IDE；"多角色" = 按 token 分级授权。设计原则：**默认安全（secure by default）、单点校验、危险操作硬隔离、一切留痕**。

## 1. 身份与 token

- **签发**：人在 IDE `Settings → Tools → Idectl` 签发/吊销命名 token（如 `claude-code-frontend`）。token 为随机 256bit；**密文存 IDE PasswordSafe，磁盘只落盐化 SHA-256 指纹**；`instances.json` 只写 `tokenHint`（前 6 位）不落明文。
- **属性四元组**：`{subject(可读名), role, allowedProjects(路径集合或 *), expiresAt?}`，可选工具黑名单。
- **传输**：每请求 `Authorization: Bearer <token>`（禁 query string）；Claude Code `--header` / Codex `bearer_token_env_var` 原生支持；stdio 桥经环境变量注入（符合规范 stdio 不用 OAuth 的指引）。
- **吊销与轮换**：吊销即时生效——其活动 MCP 会话失效（后续请求 401 / 会话 404 迫使重新 initialize，重新握手时再次校验）；支持 TTL。
- **无匿名访问**：服务**默认关闭**；首次开启强制签发第一枚 admin token。与 JetBrains 内建 MCP"无 token 仅 localhost"划清界限。
- **不上 OAuth 2.1 全套**：localhost 场景规范中 authorization 为 OPTIONAL；预留 `401 + WWW-Authenticate` 形状便于将来升级（企业场景增强项）。

## 2. 网络与协议防线（进程外）

| 防线 | 措施 |
| --- | --- |
| 绑定 | 仅 `127.0.0.1`，无配置可改为 0.0.0.0（远程需求让用户自行 SSH 隧道） |
| DNS rebinding | Host/Origin 白名单校验（`127.0.0.1` / `localhost`），失败 403，先于协议解码执行 |
| 认证 | Bearer 校验失败 401；网关层在 Ktor pipeline prePhase，**任何工具代码不可能在未认证时执行** |
| 会话绑定 | MCP `Mcp-Session-Id` 在 initialize 时与 token 绑定；token 吊销 → 会话失效 |
| 日志卫生 | token 永不回显；审计记 `argsDigest` 而非全参（危险操作除外，见 §6） |

## 3. 角色模型（RBAC，单调包含 admin ⊇ operator ⊇ viewer）

| 角色 | 心智 | 能做 | 不能做 |
| --- | --- | --- | --- |
| `viewer` | 只读观察者（CI 看板、日志分析 Agent） | 一切列举/读取/检索/状态查询：list_*、console_read/search、get_build_status、get_stack、get_variables、断点列举、测试结果 | 任何改变 IDE / 进程状态的操作 |
| `operator` | 干活的开发 Agent | + 构建、启动配置、**停止/重启自己启动的会话**、断点增删、单步控制、maven_sync、白名单 goal、profile 切换 | 停他人/user 的进程；求值；任意 goal；条件断点 |
| `admin` | 运维/受完全信任的 Agent | + 停止/重启**任何**会话（含人类手动启动的）、`evaluate`、条件断点/log_expression、白名单外 Maven goal、取消他人操作、审计查询、token 管理 | — |

实施要点：

1. **静态门槛 + 动态属主**双重判定：工具声明 `min_role`；资源级再查属主（见 §4）。
2. **tools/list 按角色过滤**：viewer 连接时根本看不见 `evaluate` 等工具——"不可见"优于"调用报错"（减少无效尝试与提示词污染）。
3. 越权调用 → `PERMISSION_DENIED`（附所需角色与 remediation），**并计入审计**。
4. 工具 `annotations`（readOnlyHint/destructiveHint）与角色矩阵一致，支持的客户端 UI 可再加一道人审。

## 4. 资源属主与并发语义

- **执行会话属主**：MCP 发起的会话 `owner=token.subject`；用户手动点绿色按钮的 `owner=user`。operator 只能 stop/restart/debug_control 属主为自己的会话；操作他人或 user 的会话一律 admin + 强制审计。
- **构建**：每项目 FIFO 队列（深度 4，满则 BUSY）；取消仅发起者或 admin；队列状态全量可见（`get_build_status` 报"谁在建、谁在等"）——透明本身就是多 Agent 协作的治理手段。
- **单实例配置冲突**：结构化 `CONFLICT` 返回，绝不弹平台模态框（模态框 = 无人值守场景的拒绝服务）。
- **项目隔离**：token.allowedProjects 之外的项目完全不可见（列表过滤 + 句柄查找限定在授权项目的注册表内）；A 项目的 Agent 看不到也停不了 B 项目的会话。

## 5. 危险操作硬隔离（三条 RCE 通道）

| 通道 | 为什么危险 | 管控 |
| --- | --- | --- |
| `evaluate`（表达式求值） | 在目标 JVM 内执行任意代码 | admin 专属 + **策略三档**：`off` / `readonly`（正则白名单拒绝赋值与方法调用副作用，默认档）/ `unrestricted`；每次调用全量审计（含表达式原文）；可选 elicitation 人工确认 |
| 断点 `condition` / `log_expression` | 同样是调试器求值的可执行代码 | 含此二参数的 breakpoints 调用升 admin |
| `maven_run_goal` 白名单外 | `exec:java`、任意插件 goal ≈ 宿主任意命令执行 | 白名单（clean/validate/compile/test/package/verify）内 operator；白名单外 admin；白名单在设置页可调 |

另：`stop_session` 对 `owner=user` 的进程（人正在手工调试的服务）定义为危险操作 → admin。

## 6. 审计

- **格式**：JSONL，每条 `{ts, subject, role, mcpSessionId, tool, project, argsDigest(SHA-256 前 12 位 + 关键字段白名单明文如 configName/goals), resultCode, durationMs}`。
- **危险操作**（evaluate / 白名单外 goal / 停他人进程 / token 变更）额外记**全量参数**。
- **不记录**控制台输出正文（体积失控 + 可能含业务机密）。
- 拒绝事件（403/配额/冲突）同样记录——越权尝试本身是重要信号。
- 落盘 `~/.idectl/audit/`（目录权限 0600），按天滚动、默认保留 30 天（可配）；admin 经 `get_audit_log` 只读查询，**只可查不可删**；token 签发/吊销本身也入审计。
- IDE 状态栏 widget：实时显示"当前连接 Agent 数 + 最近危险操作"，人可一键断开全部会话。

## 7. 配额与资源防护

| 维度 | 默认值 |
| --- | --- |
| 每 token 并发 in-flight 工具调用 | 8 |
| 每 token 活动 MCP 会话 | 4 |
| 每 token 存活运行会话（由其启动） | 8 |
| 速率 | 30 req/s |
| 控制台缓冲 | 每会话 10k 行 & 8MB；全局 128MB LRU；终止会话 TTL 30min |
| grep | 500ms deadline；pattern ≤512 字符；上下文 ≤20 行；maxMatches ≤1000 |
| evaluate | 每分钟 ≤30 |
| 长等待 | 一律带 timeout，超时转句柄（防协程池饥饿） |

超限返回结构化 `QUOTA_EXCEEDED`（附建议），Agent 可自适应退避。

## 8. 威胁模型速览

| 威胁 | 对策 |
| --- | --- |
| 本机恶意进程直连端口 | Bearer 必需 + 默认关闭 + Origin 校验 |
| 浏览器页面 DNS rebinding 打 localhost | Host/Origin 白名单 |
| token 从客户端配置文件（.mcp.json 等）泄露 | 最小角色/最小项目集签发 + TTL + 一键吊销 + 审计溯源；文档建议 env 注入 |
| 失控 Agent 循环轰炸 | 速率限制 + 配额 + 状态栏可见 + 一键断开 |
| 恶意/劣质正则 ReDoS | deadline-CharSequence 500ms 熔断 |
| 求值/goal 越权执行代码 | §5 三通道硬隔离 |
| 缓冲撑爆 IDE 堆 | §7 三级上限 + LRU + TTL |
| 审计日志泄密 | 参数摘要为主、正文不记、目录 0600 |

## 9. 渐进落地（与 09 路线图对齐）

- **M0/M1**：单枚 token（隐含 admin）+ 仅 localhost + Origin 校验 + 审计骨架——先让核心两能力安全可用。
- **M3**：完整 RBAC（三角色、属主规则、tools/list 过滤、配额、设置页 token 管理、get_audit_log）。
- **M6**：elicitation 可选确认、状态栏 widget、审计滚动策略打磨、（可选）OAuth 形状预留验证。
